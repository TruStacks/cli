Application Build & Release

Running the application build​

During the application creation, the concourse ci driver configured a pipeline for us to build the react-tutorial application.

Next we will run the application build in concourse.

First, navigate to concourse and confirm that the react-tutorial pipeline was created.

You should see the react-tutorial application in the browser.

Next, click the blank gray square in the pipeline widget to navigate to the pipeline builds page.

Run the build​

Click the "+" plus button at the top right to run the application build.

The build will start and you will see a series of resources in the browser.

Once the task: build widget turns yellow, click on it to expand the build output

Dagger & CI/CD tooling​

Dagger​

TruStacks uses Dagger for CI tasks.

Dagger abstracts CI execution tasks out of the CI tool which enables TruStacks to support any CI/CD provider.

Individual "pipeline" tasks are divided into Dagger actions. The complete list of actions that are run during the react-tutorial application build are here.

CI/CD tooling​

TruStacks interacts with two types of software delivery tools during the build process:

  • runtime tools
  • toolchain components

Runtime tools​

Are tools who's lifecycle begins and ends during the application build. These components are ephemeral and do not have long-lived deployments in the toolchain.

Ex. eslint, trivy, kustomize

Toolchain Components​

Are tools that are deployed in the toolchain and and who's lifecycle exists beyond the application build.

Ex. Argo CD, SonarQube, Pact
